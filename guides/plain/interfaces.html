Terms

Classes and objects

To start with, we need to look at classes (and objects). Interfaces in java are basically worthless without them. This isn‚Äôt meant to be a full introduction to classes, it‚Äôs just to set the stage.

So- uh- what‚Äôs the difference between an object and a class? I‚Äôll clarify because I‚Äôll use that throughout the guide.

You might see on the interwebs classes being described as the blueprint for objects, and objects being things built from those blueprints. Here‚Äôs a more practical example.

A class is‚Ä¶ the code that starts with class Name {. So the following is a class:

class Thing {
    public int field;

    Thing(int num) {
        field = num;
    }

    public int method(int argument) {
        return field + argument;
    }
}

And when I do:

Thing thingName = new Thing(57);

What is created by new Thing and then stored into thingName is an object. The creation of an object is also called instantiation. (You may also see objects referred to as instances of class, but I won‚Äôt be saying that here to avoid confusion)

Method signatures

This is another term I‚Äôll be referring to. It‚Äôs basically just the name of the method, with the types and arguments.

So if I have a method that looks like:

void coolStuff(int epicNumber) {
    UI.println("Your number is " + epicNumber);
}

The method signature would be this bit:

void coolStuff(int epicNumber)

That‚Äôs all! Here‚Äôs another example. For this method:

String moreCoolStuff() {
    return "Cool stuff";
}

The signature would be:

String moreCoolStuff()

How do you define an interface?

Let‚Äôs get right into it. Here‚Äôs how you make one:

interface NameHere {
    void someMethod(int arg);
    int otherMethod();
}

It‚Äôs similar to how you define a class, except that you use the keyword interface instead. The key difference is that the methods are just signatures! They have nothing in between {}. They don‚Äôt even have the curly brackets!

That‚Äôs all an interface is. It‚Äôs just a named thing with a list of methods. Pretty funky aye?

But like, how do you use them?

Let‚Äôs define a simple interface to do stuff with. I‚Äôll call it Talk and list a couple methods. You might have some questions as to why use interfaces, but just bear with me.

interface Talk {
    void speak(String message);
    void yell();
}

Okay, very cool, we‚Äôve got an interface. What now‚Ä¶

Let‚Äôs say you now want to create a Talk variable called speaker. You might try to do something like this:

Talk speaker = new Talk();

However, this will not compile! You will likely see a message similar to this:

Main.java:3: error: Talk is abstract; cannot be instantiated
    Talk speaker = new Talk();
                   ^
1 error

What does this even mean? Well, we can see that the error is where we tried to create a new Talk(). It did not complain about us defining a variable with the type Talk, so that‚Äôs okay. The important bit of the error is where it says ‚ÄúTalk is abstract‚Äù.

In java (and many other languages, for example: C++), something being abstract basically means that it doesn‚Äôt have an implementation. In other words, it‚Äôll have a name, but no actual code describing what it actually does! If this sounds confusing, here‚Äôs a little example.

A normal method might look like this:

void say(String message)  /* <-- This is the signature */
{                                    /* <--                             */
    UI.println("I say: " + message); /* <--  This is the implementation */
}                                    /* <--                             */

So, if we make say an abstract method, it‚Äôd look like this:

void say(String message);  /* <-- This is the signature */
A bit more info for the curious (and for the pedants <3) Actual abstract methods in java use the abstract keyword. So it‚Äôd actually look like abstract void say(String message);, and it‚Äôd be part of an abstract class instead of just a class. I chose to skip that to avoid throwing out too many things once.

So, it‚Äôd be just the method signature, right? And- hold on- that‚Äôs exactly what we put on the interface! That‚Äôs the reason why we can‚Äôt just create an object from an interface, the methods don‚Äôt have any implementation (code). Java would not know what to do at all if you called those methods.

Here‚Äôs the important bit about interfaces, the methods are there so that classes can implement them- not the interface. A class that implements those methods (i.e.¬†has methods with the same signatures as the interface) can tell java that it, in fact, implements the interface. So, if a class implements Talk, java will happily let you put objects of that class into a variable of type Talk.

Worry not, for I have more examples!

Here‚Äôs the interface again, I copied and pasted it so that you don‚Äôt have to scroll up.

interface Talk {
    void speak(String message);
    void yell();
}

So, we need to create a class that implements the interface. Let‚Äôs start by making a normal class first. I‚Äôll make a cat because my cat really likes to talk. This Cat class will be very simple, it‚Äôll just have a name, and it‚Äôll sleep.

class Cat {
    private String name;

    Cat(String name) {
        this.name = name;
    }

    void sleep(int hours) {
        for (int i = 0; i < hours; i++) {
            UI.println("zzzzzzzzzzzzzz");
        }
    }
}

Okay! We have a cat now! Let‚Äôs make it talk! I‚Äôll go step by step.

The first thing you‚Äôll want to do is to modify the class signature. (It just means changing class Cat to something else). We have to tell java that we‚Äôll be implementing the methods of the Talk interface. If we don‚Äôt, it won‚Äôt know what we‚Äôre trying to do. The change is quite straightforward.

class Cat implements Talk {
    private String name;

    Cat(String name) {
        this.name = name;
    }

    void sleep(int hours) {
        for (int i = 0; i < hours; i++) {
            UI.println("zzzzzzzzzzzzzz");
        }
    }
}

Very nice! Something of note is that if we try to compile this as it is now, java will refuse. Take a look:

Cat.java:1: error: Cat is not abstract and does not override abstract method yell() in Talk
class Cat implements Talk {
^
1 error
Further explanation of the error message We can see that it calls yell an abstract method. As we saw before, methods in interfaces are abstract by default. It also says that Cat is not an abstract class (yeah, those are a thing). Therefore, it expects us to actually implement those methods.

This is because we haven‚Äôt implemented any of the methods listed on the interface. Java isn‚Äôt happy that we lied to it about implementing Talk :(

Let‚Äôs create those methods! We‚Äôll leave them empty at first just to try it out. Something of note is that those interface methods have to be public. It makes sense when you think about it, why tell me you have some methods if I can‚Äôt use them?

class Cat implements Talk {
    private String name;

    Cat(String name) {
        this.name = name;
    }

    void sleep(int hours) {
        for (int i = 0; i < hours; i++) {
            UI.println("zzzzzzzzzzzzzz");
        }
    }

    public void speak(String message) {

    }

    public void yell() {

    }
}

If we try to create a new Cat now and put it into a variable of type Talk, you‚Äôll find that java is okay with it!

Talk speaker = new Cat("Sparrow Spanner");

We implemented the methods that we promised java we would have, so it‚Äôs happy to let us do our own thing. Before moving on, let‚Äôs write some code so that those methods actually do something.

class Cat implements Talk {
    private String name;

    Cat(String name) {
        this.name = name;
    }

    void sleep(int hours) {
        for (int i = 0; i < hours; i++) {
            UI.println("zzzzzzzzzzzzzz");
        }
    }

    public void speak(String message) {
        UI.println("I am " + name +", I guess. I say: " + message);
    }

    public void yell() {
        UI.println("MRRRAAAAAAAWWW");
    }
}

So what can we actually do with this speaker variable? Let‚Äôs try calling some methods! We‚Äôll start by the methods from the interface (speak and yell).

// Prints: I am Sparrow Spanner, I guess. I say: hi!
speaker.speak("hi!");

// Prints: MRRRAAAAAAAWWW
speaker.yell();

Nice! That‚Äôs what we expected. What if I we try to be sneaky and call a method that is specific to Cat? Let‚Äôs try and make our cat sleep for a couple hours.

// Sneaky time
speaker.sleep(20);

Java is not very happy about our trickery:

Cat.java:24: error: cannot find symbol
    speaker.sleep(20);
           ^
  symbol:   method sleep(int)
  location: variable speaker of type Talk
1 error

If we break down the error, java is saying that this variable of type Talk doesn‚Äôt have a method called sleep. And well‚Ä¶ this is true. The interface Talk doesn‚Äôt specify a method called sleep. So if a variable is type Talk, then you can only call the methods specified in the interface Talk. It doesn‚Äôt matter that we put a Cat into it, we told java it was a Talk, so we can only use Talk things.

If we have just one class in a program, the interface doesn‚Äôt seem that useful. However they come in handy once you have multiple classes in your project.

Let‚Äôs make a bunch of classes that implement interface in addition to our Cat. I‚Äôll make a Person and a Loudspeaker. If it helps you to have a context, imagine we have a file with some info, and we create one of these object depending on what the file says.

I‚Äôll copy and paste the interface code again for reference.

    void speak(String message);
    void yell();

This example does mean that I have to make more classes, the code blocks might seem big but that‚Äôs just because java is pretty verbose. I won‚Äôt really explain the classes, they‚Äôre pretty simple. However- if you want me to, just send me a message and I‚Äôll add more stuff.

Here‚Äôs our Person:

class Person implements Talk {
  String firstName;
  String lastName;
  String job;

  Person(String firstName, String lastName, String job) {
    this.firstName = firstName;
    this.lastName = lastName;
    this.job = job;
  }

  // Interface methods
  public void speak(String message) {
    UI.println("Hello I'm " + firstName + " " + lastName + " and I'm the CEO of " + job + ". I say that: " + message);
  }
  public void yell() {
    // We can use our own variables when implementing the method, that's okay!
    UI.println("RAAAAAAAAAAAGGGGGHHHHH, I HATE " + job.toUpperCase());
  }
}

I won‚Äôt- like- code a loop that reads from a file and creates objects based on that. That is left as an exercise to the reader ;). I‚Äôll just create the objects manually.

Let‚Äôs make some of these objects! We‚Äôll just have an ArrayList with the type Talk to keep track of all of them.

ArrayList<Talk> talkObjects = new ArrayList<>();
// Because all these objects implement Talk, we can put them in the ArrayList

This compiles fine! Let‚Äôs actually use the interface methods. First we‚Äôll make them all speak.
